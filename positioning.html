Css positioning

When browser parses html page it positions elements according to normal flow which means that block elements start rendering from new line and inline elements are displayed next to the previous element on the same line.
CSS gives us the possibility to change default element position to place element where we want it to be.
This is done by using css position property: 

#elementId {
    position: relative;
}

There are 4 types of values position property of an element can accept:

absolute

absolutely positioned elements are completely removed from the flow of containing block .
What it means is that now position of an element has to be specified directly, otherwise element would overlap with preceding and following elements of the container.
Element position is specified by top, bottom, left, right properties.
Note: What would be the "containing block" for absolute positioned element? Well, it would be the first element up the elements tree with position property set to relative. If such element is not found - it will be the body element.
It means that without any parent elements with relative positioning top: 0; will mean "top of the page" otherwise it would be a top of a parent element.

Example of absolute positioning: 


Some content  

    
    
#absolutelyPositioned {
    position: absolute;
    top: 5px;
    left: 0;
    border: 1px solid gray;
}

relative

Relative position property means position relative to it's position in normal flow.
First space is reserved for the element in page flow and then the element is moved using the offsets top, bottom, left, right
top: 5px; means "5px from the top of it's original position".
It is important to remember that element is not taken out of flow so when moved - it will create a "empty space" in the document.

Lets see an example:

Some content  



Bottom div
    
    
#top-1 {
    background-color: red;
}

#bottom-1 {
    background-color: blue;
}

#relativelyPositioned {
    position: relative;
    top: 5px;
    height: 100px;
    border: 1px solid gray;
    background: green;
    opacity: 0.5;
}

fixed

fixed positionmeans element is fixed in browser viewport using top, bottom, left, right
properties. It's similar to the absolute position in that element is taken out from normal page flow but the difference is that in case of fixed it's positioned against the browser window, not the document. It means that if you scroll down the page it will stay in the same place. 


Fixed 

    
    
#fixedPositioned {
    position: fixed;
    top: 20px;
    left: 20px;
    border: 1px solid gray;
    background-color: magenta;
}

You should see magenta colored box in left top corner of your browser.

inherit and static

This is the property value you won't be using often. This just means normal element position in page flow. All created elements get this by default.
The only reason to specify it is when you want to override previously set position.

Float property

float property is one of the most commonly used positioning properties in web design.
Float allows to push element to the right or to the left of the container and let other elements wrap around it.
float property accepts right and left values. Example:

    float: left;
    border: 1px solid gray;
    height: 40px;
}

Most intuitive use of floated elements is for making text wrap around the image.

float property is very popular when creating designs with columns. By making 2 divs float we can position them right next to each other.
Or by floating element to the sides of the container we can put text content in the middle width of which will be automatically adjusted when container width changes.

"Clearing" the floats

As already pointed out element that comes next to the floated element is rendered right next to it.
Let's consider 2 column design with footer. If 2 columns are floated footer would be rendered next to the second column (if width of the page allows).
This is clearly not what we wanted to do.
To remedy this we can use clear property and apply it to the next element after the floated.
This is most commonly done with empty div approach. Div with no content is created right after the floated element and property clear is applied to it.

<div style="clear: both;"></div>

Notice that style is applied directly to the div as opposed to writing it in stylesheet. The reason is simple - this element sole purpose of existence is to clear float for next elements, so the only way clear: both; is going to be changed if when element is removed, so there is no point of moving it into stylesheet file.
