Content property

One of the very interesting but not so often used CSS features is "content" property.
Lets see how it looks like: 

    .password:before {
        content: "Password: ";
    }
Now all elements with class "password" will have Password: text preceding them.

Separation Of Concerns problem

It can be debated whether this kind of thing should be handled in CSS.
Separation of concerns means that content should be handled by html and design by css.
This is the reason content property is so rarely used.

In some cases using content property can be a design decision.
Consider 2 versions of the same page - normal and mobile version.
For mobile version we would display small key image before password field
For normal version we would output "Password:"
As you can see this is clearly a design decision, so use of content property is well justified.

Checkmark example:

Let's see one of content property example: 

#hoverExample a:hover:before {
   content:  "\2713 ";
}
"\2713 " here is an ASCII character code for checkmark symbol.
This styling will add checkmark symbol before the link you are hovering upon.
