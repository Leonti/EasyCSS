Cascading and media types

Cascading

Cascading Style Sheets cascade. This means that the styles are applied in order as they are read by the browser. The first style is applied and then the second and so on. What this means is that if a style appears at the top of a style sheet and then is changed lower down in the document, the second instance of that style will be the one applied, not the first.

The document tree is linked to inheritance of property styles. Elements lying lower in the hierarchy (descendants) inherits attributes from ancestors, when they don't have specified css rules.

This paragraph is blue, inside there is italics and bold, which have not been given any coloring. So they inherit color from the ancestor (in this case from paragraph), but have font-weight and font-style set as their own styles. There is also a span which has been given following attributes: color: green;. Color of the text is not inherited - it's overwritten: span.

The !important rule is a way to make your CSS cascade but also have the rules you feel are most crucial always be applied. A rule that has the !important property will always be applied no matter where that rule appears in the CSS document. So if you wanted to make sure that a property always applied, you would add the !important property to the tag. So, to make the paragraph text always blue, in the above example, you would write: 

    p { color: blue !important; }

Media types

One of the key advantages of styles is possible to determine how the document will look in different media: on screen, paper, speech synthesizer, a device for reading braille, etc.

There are two ways to determine the independence of style sheets and assign different attributes to different media:

Specifying the target medium in a style sheet, using the command @media or @import:

	@media print {
        /* CSS for print */
	}

        @import url(print.css) print;

Specifying the target medium within the html document

        <!DOCTYPE>
        <html>
        <head>
            <title>Target medium</title>
            <link rel="Stylesheet" type="text/css" media="print, handheld" href="print.css" />
        </head>
        <body>
            <p>Content</p>
        </body>
        </html>

Media Types

The names of the media are a reflection of target devices for which the relevant properties make sense (CSS 2):

all - for all devices
aural - speech synthesizers (auditory style sheets) - CSS 2.1 introduces instead the type of speech
braille - devices for reading braille
embossed - braille Printers
handheld - hand-held devices (typically small screen, monochrome, limited bandwidth)
print - opaque printed materials and print preview
projection – for pojectors
screen - color computer screen
tty - media using a grid of fixed width as teleprinters, terminals, or portable devices with limited display capabilities
tv – for tv
